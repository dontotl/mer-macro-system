# 텔레그램 매매신호 / 유닛관리 알림 설계서

작성일: 2026-04-01
목적: 실제 매매는 MTS에서 수행하고, 시스템은 종목/유닛 상태관리와 매매신호 알림만 제공한다.

## 1. 운영 원칙
- 시스템은 **자동매매를 하지 않는다**.
- 사용자는 MTS에서 직접 주문한다.
- 시스템은 아래만 담당한다:
  - 신규 진입 후보 탐지
  - 추매(+8%, +16%) 신호 탐지
  - 손절 / 20일선 이탈 알림
  - 현재 보유 종목 / 유닛 / 기준가 상태 요약
- 기준 신호는 가급적 **종가 기준 확정**으로 운용한다.
- 장중 알림은 참고용(근접 알림)으로 분리한다.

## 2. 전략 기준 요약
### 종목 선정
- KOSPI + KOSDAQ
- 시총 1조 이상
- RS 상위 10%
- 3개월 신고가
- 5/10/20/60일 정배열
- 최초 진입 시:
  - 양봉
  - 당일 거래대금 >= 직전 3거래일 평균 거래대금의 1.5배

### 포지션 규칙
- 총계좌 5분할
- 1유닛 = 총자산의 20%
- 종목당 최대 3유닛
- 최초 진입: 1유닛
- 추가매수:
  - 최초 진입가 대비 +8% -> 1유닛 추가
  - 최초 진입가 대비 +16% -> 1유닛 추가

### 청산 규칙
- 종가가 20일선 이탈 -> 전량매도 신호
- 손절:
  - 1유닛 보유: 1유닛 진입가 대비 -8%
  - 2유닛 보유: 2유닛 진입가 대비 -8%
  - 3유닛 보유: 3유닛 진입가 대비 -8%

## 3. 알림 종류
### A. 신규 진입 알림
트리거:
- 종가 기준 모든 진입 조건 충족

메시지 예시:
- [신규진입] 삼성전자
- 현재가: 112,000원
- 사유: RS10 / 3개월 신고가 / 정배열 / 거래대금 급증 양봉
- 권장 액션: 1유닛 진입 검토

### B. 추매 알림
트리거:
- 보유 종목이 최초 진입가 대비 +8% / +16% 도달

메시지 예시:
- [추매신호] SK하이닉스
- 현재 보유: 1유닛
- 현재가: 691,000원
- 최초 진입가 대비 +8% 도달
- 권장 액션: 1유닛 추가 검토

### C. 손절 알림
트리거:
- 마지막 유닛 기준 -8% 하락

메시지 예시:
- [손절신호] NAVER
- 현재 보유: 2유닛
- 손절 기준가 이탈
- 권장 액션: 전량매도 검토

### D. 20일선 이탈 알림
트리거:
- 종가 기준 20일선 하향 이탈

메시지 예시:
- [추세이탈] 삼성물산
- 종가가 20일선 하회
- 권장 액션: 전량매도 검토

### E. 보유 종목 요약
주기:
- 장마감 후 1회
- 또는 사용자가 요청 시 즉시

메시지 예시:
- [보유현황]
- 삼성전자 / 2유닛 / 최초진입 112,000 / 추매1 121,000 / 손절기준 111,320
- SK하이닉스 / 1유닛 / 최초진입 640,000 / 손절기준 588,800
- 현금여력: 2유닛

## 5. 장중 vs 종가 운영
### 장중
- 근접 알림만 허용
- 예:
  - 추매가 1% 이내 접근
  - 손절가 1% 이내 접근
  - 신규 진입 후보 실시간 감지

### 종가
- 최종 확정 알림
- 실제 매매 판단 기준은 종가 시그널 우선

## 6. 알림 스케줄 권장안
- 09:00~15:20: 장중 10~15분 간격 스캔 (선택)
- 15:25~15:40: 장마감 확정 신호 계산
- 15:40 이후: 최종 알림 발송

## 7. 사용자 액션 흐름
1. 텔레그램 알림 수신
2. MTS에서 종목/가격 확인
3. 직접 주문
4. 체결 후 시스템 상태 업데이트
   - 수동 반영 또는 간단 명령으로 반영

## 8. 권장 명령 구조
- `/holding` : 현재 보유 종목 보기
- `/signal` : 오늘 신규 진입 후보 보기
- `/close` : 오늘 종가 기준 매도 신호 보기
- `/filled 005930 buy 1 112000` : 체결 반영
- `/filled 005930 add 1 121000` : 추매 반영
- `/filled 005930 sell all 118500` : 전량매도 반영

## 9. 구현 단계 제안
1. 종가 기준 신호 생성기
2. 보유 상태 JSON 관리기
3. 텔레그램 알림 포맷터
4. 체결 반영 명령 처리기
5. 장중 근접 알림(선택)

## 10. 주의사항
- 신호와 실제 체결은 다를 수 있음
- 장중 급등/급락으로 시그널 왜곡 가능
- 실제 주문은 반드시 사용자가 최종 확인
- 시스템은 보조 의사결정 도구로 사용
